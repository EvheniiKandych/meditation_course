## 1. What runs this site? (Technology Stack)

The modern web space is built from several layers. Our project uses the most modern and popular stack:

*   **React (v19):** This is a library for building user interfaces. Instead of writing one huge HTML document, React allows you to create small "building blocks" (components), such as `<Header />` or `<Logo />`, and assemble the page from them.
*   **Next.js (v15+):** This is a framework on top of React. React by itself only draws buttons. Next.js gives us routing (transitions between pages), loading speed optimization (SSG/SSR), and SEO.
*   **TypeScript:** This is the same JavaScript, but with a "strict supervisor". It checks our code for errors *before* we launch the site. For example, if we say a button accepts text, but we pass it a number, TypeScript will stop us.
*   **Tailwind CSS:** A modern way of styling. Instead of writing separate CSS files, we use "utility classes" right in the HTML (for example, `text-center`, `mt-8`, `bg-bg`).

## 3. How to work with the code: Main rules of a Senior developer

### Rule 1: "Divide and Conquer"
Never write everything in one file. If you see that a page is becoming too big (more than 150-200 lines), it means some part of it should be moved to a separate file in the `components/` folder.
That's why our `page.tsx` looks so clean: it only calls `<Hero />`, `<Method />`, `<FAQ />`.

### Rule 2: Single Source of Truth
*   **For texts:** If you need to change a question in the FAQ, don't look for it in the `FAQ.tsx` layout. Open `data/content.ts` and change it there. The component will pull the new data itself.
*   **For design (colors/fonts):** Don't write the color `#F5E4DA` right in the component. Add it to `tailwind.config.ts` (as we did with the logo colors `brand-o`), and use the class `text-brand-o`.

### Rule 3: Mobile First
When you write classes in Tailwind, the base class is always applied to mobile devices. For larger screens, use prefixes:
*   `text-[32px]` — size for mobile phones.
*   `sm:text-[40px]` — size for tablets (from 640px width).
*   `md:text-[50px]` — size for desktops (from 768px width).

## 5. Practice: How to add a new section to the site? (Algorithm of actions)

Imagine you need to add an "About Author" section. Here's how you should act:

1.  **Create a component:**
    Create the file `components/sections/AboutAuthor.tsx`.
2.  **Write the basic structure (Boilerplate):**
    ```tsx
    import type { ReactNode } from "react";
    import { SectionTitle } from "@/components/ui/SectionTitle";

    export function AboutAuthor(): ReactNode {
      return (
        <section className="mb-20">
          <SectionTitle top="About" accent="Author" />
          <p className="mt-4 text-ink/80 text-sm">Text about the author will be here.</p>
        </section>
      );
    }
    ```
3.  **Connect the component:**
    Open `app/page.tsx`, import the new component at the top (`import { AboutAuthor } from "@/components/sections/AboutAuthor";`) and paste `<AboutAuthor />` where you want to see it (for example, before `<FAQ />`).

## 6. Working with AI (Helper Prompts)

Since you are just learning, AI is your best friend. But for it to give good code, it needs to be given the right commands (prompts).

If you encounter something new, copy one of these prompts and send it to me (or another AI):

**Prompt for creating a new component by design:**
> "Hi! I want to create a new React component for Next.js (with TypeScript and Tailwind CSS). It will be a 'Testimonials' section. It should contain a title (use our SectionTitle component) and a grid with 3 cards. Each card has a white background, a shadow (shadow-soft), the client's name, and the text of the review. Please generate the code for the `components/sections/Testimonials.tsx` file and show what data needs to be added to `data/content.ts`."

**Prompt for solving an error:**
> "I got an error in the terminal when launching Next.js. Here is the error text: [INSERT ERROR TEXT]. Explain to me, like a Senior, why it occurred, and show which file and how exactly it needs to be changed to fix it. Don't just give the code, tell the logic."

**Prompt for complex logic (e.g., submitting a form):**
> "I need to make it so that when clicking the 'Request Access' button in the `AccessForm.tsx` file, the data from the form doesn't just reload the page, but is collected in JSON format. Show how to convert this component to 'use client', add an `onSubmit` handler using React, and output the collected data (name, email, focusLevel) to console.log."

## 1. На чому працює цей сайт? (Стек технологій)

Сучасний веб-простір будується з кількох шарів. Наш проєкт використовує найсучасніший та найпопулярніший стек:

*   **React (v19):** Це бібліотека для побудови інтерфейсів. Замість того, щоб писати один величезний HTML-документ, React дозволяє створювати маленькі "деталі конструктора" (компоненти), як-от `<Header />` чи `<Logo />`, і збирати з них сторінку.
*   **Next.js (v15+):** Це фреймворк поверх React. React сам по собі лише малює кнопочки. Next.js дає нам маршрутизацію (переходи між сторінками), оптимізацію швидкості завантаження (SSG/SSR) та SEO.
*   **TypeScript:** Це той самий JavaScript, але із "суворим наглядачем". Він перевіряє наш код на помилки *до* того, як ми запустимо сайт. Наприклад, якщо ми скажемо, що кнопка приймає текст, а передамо їй число, TypeScript нас зупинить.
*   **Tailwind CSS:** Сучасний спосіб стилізації. Замість того, щоб писати окремі CSS-файли, ми використовуємо "класи-утиліти" прямо в HTML (наприклад, `text-center`, `mt-8`, `bg-bg`).

## 2. Анатомія проєкту: Що і де лежить?

Відкрий папку `C:/work/meditation_course` у своєму редакторі (VS Code, Cursor, WebStorm). Ось головні папки, з якими ти будеш працювати:

```text
meditation_course/
├── app/                  # Серце Next.js. Тут лежать наші сторінки.
│   ├── globals.css       # Глобальні стилі сайту (сюди ми підключали шрифти).
│   ├── layout.tsx        # "Обгортка" для всіх сторінок (тут підключаються шрифти і мета-теги).
│   └── page.tsx          # Головна сторінка нашого сайту (компонує всі секції).
│
├── components/           # Наші деталі конструктора (React-компоненти).
│   ├── sections/         # Великі блоки (Шапка, Підвал, Секція FAQ, Форма).
│   └── ui/               # Маленькі, перевикористовувані деталі (Логотип, Заголовок секції).
│
├── data/                 # База даних контенту.
│   └── content.ts        # Усі тексти (опис курсу, питання FAQ). Щоб змінити текст, йди сюди.
│
├── tailwind.config.ts    # Головні налаштування дизайну (твої кольори, шрифти, відступи).
└── package.json          # Паспорт проєкту (тут записані всі встановлені бібліотеки та команди для запуску).
```

## 3. Як працювати з кодом: Основні правила Senior-розробника

### Правило 1: "Розділяй і володарюй"
Ніколи не пиши все в одному файлі. Якщо ти бачиш, що сторінка стає завеликою (більше 150-200 рядків), значить, якусь її частину треба винести в окремий файл у папку `components/`.
Саме тому наш `page.tsx` виглядає так чисто: він лише викликає `<Hero />`, `<Method />`, `<FAQ />`.

### Правило 2: Єдине джерело істини (Single Source of Truth)
*   **Для текстів:** Якщо тобі треба змінити питання в FAQ, не шукай його у верстці `FAQ.tsx`. Відкрий `data/content.ts` і зміни його там. Компонент сам підтягне нові дані.
*   **Для дизайну (кольорів/шрифтів):** Не пиши колір `#F5E4DA` прямо в компоненті. Додай його в `tailwind.config.ts` (як ми зробили з кольорами для логотипа `brand-o`), і використовуй клас `text-brand-o`.

### Правило 3: Адаптивність перш за все (Mobile First)
Коли пишеш класи в Tailwind, базовий клас завжди застосовується до мобілок. Для більших екранів використовуй префікси:
*   `text-[32px]` — розмір для мобільних телефонів.
*   `sm:text-[40px]` — розмір для планшетів (від 640px ширини).
*   `md:text-[50px]` — розмір для десктопів (від 768px ширини).

## 4. Як запустити сайт і побачити зміни?

Якщо ти хочеш побачити свій сайт у браузері, тобі потрібно:
1. Відкрити термінал (в Android Studio або VS Code це зазвичай внизу екрана).
2. Переконатися, що ти в папці `C:/work/meditation_course`.
3. Написати команду: `npm run dev` (або `yarn dev` / `pnpm dev`, залежно від того, що ти використовуєш).
4. Відкрити браузер за посиланням: `http://localhost:3000`.

*Коли ти зберігаєш зміни в коді (Ctrl+S / Cmd+S), сторінка в браузері оновиться автоматично.*

## 5. Практика: Як додати нову секцію на сайт? (Алгоритм дій)

Уяви, що тобі треба додати секцію "Про автора" (About Author). Ось як ти маєш діяти:

1.  **Створи компонент:**
    Створи файл `components/sections/AboutAuthor.tsx`.
2.  **Напиши базову структуру (Boilerplate):**
    ```tsx
    import type { ReactNode } from "react";
    import { SectionTitle } from "@/components/ui/SectionTitle";

    export function AboutAuthor(): ReactNode {
      return (
        <section className="mb-20">
          <SectionTitle top="About" accent="Author" />
          <p className="mt-4 text-ink/80 text-sm">Тут буде текст про автора.</p>
        </section>
      );
    }
    ```
3.  **Підключи компонент:**
    Відкрий `app/page.tsx`, імпортуй новий компонент вгорі (`import { AboutAuthor } from "@/components/sections/AboutAuthor";`) та встав `<AboutAuthor />` туди, де хочеш його бачити (наприклад, перед `<FAQ />`).

## 6. Робота з ШІ (Промпти-помічники)

Оскільки ти тільки вчишся, ШІ — твій найкращий друг. Але щоб він давав хороший код, йому треба давати правильні команди (промпти).

Якщо ти зіткнувся з чимось новим, скопіюй один із цих промптів і надішли мені (або іншому ШІ):

**Промпт для створення нового компонента за дизайном:**
> "Привіт! Я хочу створити новий React компонент для Next.js (з TypeScript та Tailwind CSS). Це буде секція 'Відгуки' (Testimonials). Вона має містити заголовок (використай наш компонент SectionTitle) та сітку (grid) з 3 карток. Кожна картка має білий фон, тінь (shadow-soft), ім'я клієнта та текст відгуку. Будь ласка, згенеруй код для файлу `components/sections/Testimonials.tsx` та покажи, які дані треба додати в `data/content.ts`."

**Промпт для вирішення помилки:**
> "У мене виникла помилка в терміналі при запуску Next.js. Ось текст помилки: [ВСТАВ ТЕКСТ ПОМИЛКИ]. Поясни мені, як Senior, чому вона виникла, і покажи, який файл і як саме потрібно змінити, щоб її виправити. Не давай просто код, розкажи логіку."

**Промпт для складної логіки (наприклад, відправка форми):**
> "Мені потрібно зробити так, щоб при натисканні на кнопку 'Request Access' у файлі `AccessForm.tsx` дані з форми не просто перезавантажували сторінку, а збиралися у форматі JSON. Покажи, як перетворити цей компонент на 'use client', додати обробник `onSubmit` за допомогою React, і вивести зібрані дані (ім'я, email, focusLevel) у console.log."
